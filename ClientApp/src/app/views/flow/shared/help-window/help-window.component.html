About
Seamless webhook integration makes it easy for customers/businesses to interact with this product. This webhook must be developed and hosted by the client. Before you kick off designing and developing webhook, consider the following.
Authentication guide
ACP is supporting following authentication methods
  1. No Authentication: Nothing to do in header, just keep it blank.
  2. HTTP basic authentication:  It uses username and password and relies on Base64 encoding.
  3. API access tokens: API keys have unique identifiers for each user and for every time they attempt to authenticate.
  4. Bearer Authentication:  HTTP authentication scheme that involves security tokens called bearer tokens. 
Webhook Method & Content Type
ACP is supporting following http methods
  • GET
  • POST
Webhook Request Parameter settings
  • Clients can define their own parameter names in the request body and will be able to assign system variables, user-defined variables, and static values ​​(length limit 50) to this parameter.
  • ACP will support up to 10 input parameters in a webhook call.
  • When configuring date parameters, please note that ACP will only transmit date parameters in the format "DD/MM/YYYY HH24:MI:SS". Therefore, the client API should only be developed in this way.
Example:
  • Plain Text
  • Application/Json
Webhook Response settings
  • Clients can define their own parameter names in the response body and will be able to assign to a user-defined variables.
  • ACP will support up to 10 input parameters in a webhook call.
  • When configuring date parameters in response body, please note that ACP will only accept date parameters in the format "DD/MM/YYYY HH24:MI:SS" . Therefore, the client API should only be developed in this way.
